# 2026-05-28 设置整合 / 入职向导 / 注册功能

## 完成内容

### 1. 员工编号 → 员工ID 重命名
- `EmployeeList.vue` 表格列标题改为「员工ID」，提示文案同步更新
- `EmployeeDashboard.vue` badge 标签改为「员工ID」

### 2. 登录页注册功能
- **后端**: `auth.js` 新增 `POST /api/register` 接口
  - 注册即登录，返回 JWT token
  - 默认角色 employee，可选填写真实姓名/手机号
  - 重复账号返回错误
- **前端**: `Login.vue` 新增「注册账号」按钮 → 弹出注册弹窗（账号/密码/姓名/手机号）

### 3. AI 权限 + 用户管理 并入系统设置
- **`SystemSettings.vue`** 新增两个 tab：
  - **AI 权限 tab**: 角色×工具矩阵（开关即时保存）+ 个人例外覆盖（用户选择器 + 工具开关）
  - **用户管理 tab**: 用户表格（ID/账号/角色/创建时间/操作）+ 新增用户弹窗 + 分配角色弹窗
- **侧边栏**: 移除「用户管理」「AI 权限」菜单项（已收敛到系统设置）
- **路由**: 移除独立路由，旧路径重定向到 `/main/system/settings`

### 4. 入职向导补全岗位选择
- **`OnboardingWizard.vue`** 步骤从 4 步扩展到 5 步：
  - 欢迎 → 个人信息 → **选择岗位** → AI 偏好 → 完成
  - 岗位选择卡片式：财务 / 仓管 / 员工 / 管理员
  - 选中后展示该岗位的 AI 能力说明
- **后端**: `users.js` 的 `PUT /api/profile/onboarding` 新增 `role` 字段支持
  - 限制可选角色：finance / warehouse / employee / admin
- **提交**: `finish()` 时将 `selectedRole` 一并发送到后端

## 关键要点
- 注册用户默认角色为 employee，可在系统设置 → 用户管理 中修改
- 入职向导的岗位选择会在完成时更新用户角色
- AI权限和用户管理已被移除出侧边栏，统一归入系统设置
- 如果在其他页面开发中遇到需要删除的手，先把残留的ai权限和用户管理代码删了，比如侧边栏菜单里的，还有路由配置里的

## 验证方式
1. 打开登录页 → 点「注册账号」→ 注册后自动进入系统
2. 进入系统设置 → 看到 AI 权限和用户管理 tab
3. 系统设置 → 用户管理 → 新增/分配角色/删除正常
4. 系统设置 → AI 权限 → 切换角色开关 → 用对应角色测 AI 工具权限
5. 新用户注册后弹出入职向导 → 步骤完整，岗位选择工作正常
