# 门店交接到施工承接流程 V1

> 目的：把“门店/渠道签单，简尚负责施工交付”的真实流程固定下来，作为项目工单、仓库、班组、财务、AI 导入和总监工作区后续开发的共同基准。

## 1. 业务定位

简尚不是销售 CRM，也不是客户商城。简尚系统的核心主线是：

`外部门店/渠道交接订单 -> 简尚工程部承接施工 -> 仓库备料出库 -> 班组进场施工 -> 验收结算 -> 完工归档`

系统里的“项目工单”应该围绕施工承接，而不是围绕销售开单。

## 2. 参与角色

| 角色 | 主要职责 | 系统关注点 |
| --- | --- | --- |
| 门店/渠道/异业 | 签单、收集客户需求、交接客户与项目信息 | 不一定登录系统，信息由内部人员录入或导入 |
| 工程部/总监 | 接收交接、判断能否进场、安排施工流程 | 项目工单主负责人 |
| 勘察人员 | 到现场拍照、记录基层和整改项 | 现场勘察表、二次勘察表 |
| 仓库 | 按项目工单准备材料、出库、回库 | 出库申请、材料清单、库存扣减、余料回库 |
| 施工班组/班组长 | 进场施工、反馈现场问题、完工 | 班组安排、人员状态、工费结算 |
| 财务 | 收付款、班组费用、成本核算、利润复核 | 收款/付款/成本/利润 |
| 管理员/超级管理员 | 权限、异常处理、流程复核 | 不能绕过流程和日志 |

## 3. 标准流程

### 3.1 门店交接

输入资料：

- 来源门店/渠道
- 门店接单人
- 接单日期
- 门店单号/合同号
- 客户姓名
- 客户电话
- 省/市/详细地址
- 产品/工艺/面积/报价口径
- 是否需要施工
- 是否需要备货
- 备注与特殊要求

系统动作：

- 创建或导入项目工单草稿。
- 缺客户、电话、地址、来源时标记“缺核心资料”。
- 同手机号、客户名、门店单号命中旧工单时标记“可能重复”。
- AI 可以解析和提示，但不能直接绕过人工确认创建正式工单。

### 3.2 现场勘察

输入资料：

- 现场勘察表
- 现场照片
- 勘察日期
- 勘察人员
- 现场基层情况
- 施工面是否具备
- 需要整改的问题

系统动作：

- 记录工勘日期和工勘报告。
- 图片和 PPT 挂到项目附件。
- 形成整改项和进场判断：
  - 可进场
  - 有条件进场
  - 暂不建议进场

### 3.3 二次勘察 / 复尺

触发条件：

- 现场未达到进场条件。
- 面积、基层、柜体、门窗、踢脚线等条件发生变化。
- 总监或工程部要求复核。

输入资料：

- 二次勘察表
- 工勘复尺单
- 复核面积
- 复核后的整改结果

系统动作：

- 保存面积修正记录。
- 保留 AI/系统原始识别值和人工修正值。
- 面积、金额、地址等关键字段变化必须留日志。

### 3.4 施工交底

输入资料：

- 施工交底单
- 工勘交底单
- 成本交底单
- 施工区域
- 纹理/工艺/颜色
- 预收面积/实际面积
- 班组安排

系统动作：

- 确认施工范围和工艺。
- 确认班组长和施工负责人。
- 进入“待材料出库”。

### 3.5 材料出库

输入资料：

- 材料出库表
- 主材清单
- 辅材损耗
- 工具清单
- 出库时间
- 出库数量
- 回库数量
- 运输费

系统动作：

- 工程部发起项目出库申请。
- 仓库确认后扣减库存。
- 工单材料出库状态变为已出库。
- 后续需要扩展：回库、损耗、库存流水、成本自动归集。

### 3.6 进场施工

输入资料：

- 开工日期
- 施工负责人
- 班组长
- 施工成员
- 现场施工记录
- 异常问题

系统动作：

- 工人状态变为占用/施工中。
- 施工负责人可更新施工记录。
- 现场问题进入项目日志，必要时回到整改或复勘。

### 3.7 完工验收

输入资料：

- 完工日期
- 验收日期
- 验收照片/报告
- 质检复核表
- 客户或内部验收意见

系统动作：

- 工单进入已完工/待回库。
- 没有维修也允许完结主工程；售后应作为独立事件。

### 3.8 班组工费结算

输入资料：

- 施工班组工费结算单
- 班组人员
- 施工面积
- 单价
- 小计
- 工费合计

系统动作：

- 形成应付班组费用。
- 后续可联动财务付款记录。
- 工费变化必须留日志。

### 3.9 完工成本核算

输入资料：

- 完工成本核算表
- 工程款总金额
- 施工面积
- 人工费
- 材料费
- 辅料费
- 工具费
- 运输费
- 管理费
- 成本合计
- 毛利润
- 利润率
- 成本控制结果

系统动作：

- 形成项目利润视图。
- 自动对比材料出库、工费结算和财务流水。
- 金额口径必须区分：
  - 门店/客户销售总价
  - 简尚施工结算收入
  - 项目成本
  - 项目毛利

### 3.10 财务结算与归档

输入资料：

- 项目结算付款单
- 项目结算收款单
- 财务结算表
- 未收/未付状态
- 归档月份

系统动作：

- 关联财务流水。
- 标记已结算、未付、未收。
- 完工项目进入归档。
- 售后不阻塞主工程完结。

## 4. 系统状态建议

| 状态 | 含义 |
| --- | --- |
| 草稿/待确认 | 门店资料刚导入，等待人工确认 |
| 资料已确认 | 核心资料齐全，可以进入工勘 |
| 已勘察 | 完成现场勘察或工勘记录 |
| 待整改/待复勘 | 现场暂不满足进场条件 |
| 条件已满足 | 可安排班组 |
| 已排班 | 班组长/施工负责人/成员已安排 |
| 已交底 | 施工范围和工艺已确认 |
| 待出库 | 等仓库备料出库 |
| 已出库/待进场 | 材料已出库，可安排进场 |
| 施工中 | 班组已进场施工 |
| 待验收 | 施工完成，等待检查验收 |
| 已完工 | 验收完成，等待回库/结算 |
| 已回库 | 余料/工具处理完成 |
| 待结算 | 等财务、班组、成本核算 |
| 已完结 | 主工程闭环归档 |
| 售后中 | 独立售后事件，不阻塞主工程 |

## 5. 关键校验规则

- 没有来源、客户、电话、地址，不允许推进正式施工流程。
- 没有工勘或现场勘察，不应直接进入施工准备。
- 勘察结论为“暂不建议进场”时，需要整改或复勘记录。
- 没有班组长、施工负责人或施工成员，不允许进入交底/进场。
- 材料未出库时，不应进入实际施工。
- 完工后可以没有售后，主工程必须能正常完结。
- 金额、面积、地址、客户电话、状态推进属于关键变更，必须留日志。
- AI 只能解析、提示、生成草稿；创建、推进、改金额必须人工确认并走后端权限。

## 6. 文件与表单对应关系

| 文件/表单 | 系统模块 |
| --- | --- |
| 现场勘察表 / 基层勘察表 | 现场勘察、整改项、进场判断 |
| 二次勘察表 / 工勘复尺单 | 复勘、面积修正、整改复核 |
| 施工交底单 / 工勘交底单 / 成本交底单 | 施工范围、工艺、面积、班组交底 |
| 材料出库表 | 仓库出库、回库、材料成本 |
| 施工班组工费结算单 | 班组工费、人工成本 |
| 完工成本核算表 | 项目成本、毛利、利润率 |
| 项目结算收款/付款单 | 财务流水、收付款、未收未付 |
| 施工照片/验收照片/现场视频 | 工单附件、施工记录、验收依据 |

## 7. 后续开发顺序

1. 先稳定项目工单主流程和附件归属。
2. 做现场勘察表生成器：固定模板、图片排版、整改项、进场判断。
3. 做材料出库/回库联动：申请、确认、扣库存、回库、损耗。
4. 做班组工费结算表单：人员、面积、单价、合计。
5. 做完工成本核算：材料、人工、辅材、工具、运输、管理费、利润。
6. 做工作树一次性导入：总监旧文件夹生成待确认清单，不自动污染正式数据。
7. 做 AI 增强：识别图片、拆表格字段、发现缺项和金额冲突。

## 8. 当前 V1 边界

- 本 SOP 是业务基准，不代表所有功能已经完成。
- 先服务内部员工和总监，不做客户登录、商城、经销商系统。
- 门店人员暂时不需要登录系统。
- 工作树体检器只是一次性导入脚手架，不是长期日常模块。
- 日常模块应优先落在工程部工作台和项目工单详情。
