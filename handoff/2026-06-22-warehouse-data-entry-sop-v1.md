# 仓库数据录入 SOP V1

适用范围：简尚系统产品库存、仓库盘点导入、材料出库/回库、供货单自有库存选品、AI 仓库查询。

目标：让仓库材料从源头就按“名称、规格、仓库单位、库存、库位、提醒线”录清楚，避免同名多规格、单位混乱、库位不明导致后续出库、回库、盘点和 AI 查询出错。

## 1. 核心口径

仓库材料必须按 SKU 录入。SKU 的意思是：同一个材料只要规格不同，就算不同库存项。

正确示例：

- 霞光沙1L｜仓库单位：桶｜当前20桶｜低于3桶提醒｜单价100/桶｜库位A-1-1-1
- 霞光沙5L｜仓库单位：桶｜当前15桶｜低于3桶提醒｜单价500/桶｜库位A-1-1-2

错误示例：

- 霞光沙｜单位：L｜库存100
- 霞光沙｜库存35桶
- 霞光沙1L/5L混在同一行

解释：

- `霞光沙` 是材料名称。
- `1L / 5L / 20kg` 是规格，不是库存单位。
- `桶 / 支 / 卷 / 套 / 个` 是仓库单位。
- 系统扣库存时扣的是“桶、支、卷”等仓库单位，不扣“L”。

## 2. 必填字段

新建或导入仓库材料时，至少要确认这些字段：

| 字段 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| 产品/材料名称 | 必填 | 霞光沙 | 不带规格时也可以，但同名多规格必须单独成行 |
| 规格 | 强烈建议 | 5L、1L、20kg | 用来区分同名材料 |
| 仓库单位 | 必填 | 桶 | 库存、出库、回库都按这个单位计算 |
| 当前库存 | 必填 | 15 | 只填数字，页面会显示为 15桶 |
| 低库存提醒 | 建议 | 3 | 低于这个数量时提醒 |
| 参考单价 | 可选 | 500 | 页面显示为 500/桶 |
| 分类 | 建议 | 艺术漆、辅材、工具 | 后续搜索和筛选用 |
| 仓库编码/库位 | 建议 | A-1-1-1 | 用来对应真实仓库摆放位置 |
| 备注 | 可选 | 仅限必要说明 | 不要把临时摆放位置写进备注当库位 |

## 3. 仓库编码规则

当前采用四段式编码：

```text
A-1-1-1
```

含义：

- `A`：区域，比如 A 区、B 区、C 区。
- 第一个 `1`：1 号货架。
- 第二个 `1`：第 1 排。
- 第三个 `1`：第 1 格。

示例：

- `A-1-1-1`：A区 1号货架 第1排 第1格
- `A-2-1-1`：A区 2号货架 第1排 第1格
- `B-1-2-3`：B区 1号货架 第2排 第3格

规则：

- 字母统一大写。
- 中间用英文短横线 `-`。
- 不使用“门口左边”“墙边”“地上”“临时放置”等临时位置作为正式库位。
- 临时位置只适合现场沟通，不适合作为系统仓库编码。

## 4. 产品命名规则

页面显示建议：

```text
霞光沙5L
仓库单位：桶｜当前15桶/低于3桶提醒｜单价：500/桶｜分类：艺术漆｜库位：A-1-1-1
```

数据库拆分建议：

- `name`：霞光沙
- `spec`：5L
- `unit`：桶
- `stock`：15
- `min_stock`：3
- `unit_price`：500
- `price_unit`：桶
- `warehouse_code`：A-1-1-1
- `category`：艺术漆

如果历史表格里已经把名称和规格写在一起，例如 `霞光沙5L`，导入时先尽量拆成：

- 名称：霞光沙
- 规格：5L

如果暂时拆不准，先保留在名称里，但后续整理仓库清单时要拆开。

## 5. Excel 盘点表导入规则

当前系统盘点导入草稿支持从 Excel 中识别常见字段。

推荐字段映射：

| Excel 字段 | 系统字段 |
| --- | --- |
| 款式 规格 / 材料名称 / 产品名称 | 产品/材料名称 |
| 色号 / 规格 / 规格说明 | 规格 |
| 功能名称 / 分类 / 类别 | 分类 |
| 单位 / 仓库单位 | 仓库单位 |
| 仓库库存数量 / 当前库存 / 数量 | 当前库存 |
| 单价（元） / 单价 / 参考单价 | 参考单价 |
| 存放位置 / 仓库编码 / 库位 | 仓库编码 |
| 阀值 / 阈值 / 低库存提醒 / 最低库存 | 低库存提醒 |
| 备注 / 说明 | 备注 |

导入流程：

1. 财务或仓库上传盘点表。
2. 系统生成“盘点导入草稿”，不立刻覆盖库存。
3. 系统尝试按名称、规格、单位匹配已有产品。
4. 匹配成功的行可以确认更新库存。
5. 匹配失败的行必须人工处理，不能让系统猜第一个同名材料。
6. 确认后系统更新库存、低库存提醒和库位余额。
7. 只有库存数量变化时写库存流水；只改提醒线不写库存流水。

注意：

- Excel 前几行可能是标题，不一定第一行就是表头；导入器会尝试自动找表头。
- 用户当前样本表里出现过 `款式 规格`、`功能名称`、`仓库库存数量`、`存放位置`、`阀值`。
- `门口左边` 这类临时位置会被过滤，不写成正式库位。

## 6. 出库、回库、供货单联动规则

自有库存出库或供货时，必须选择系统里的产品，不允许只手填名称。

下拉显示口径：

```text
霞光沙5L｜桶｜库存15｜A-1-1-1
```

联动规则：

- 选择产品后自动带出产品 ID、名称、规格、分类、仓库单位、单价、库位。
- 自有库存供货必须有关联 `product_id`。
- 总部直发供货可以手填材料名，不强制关联库存。
- 同名多规格材料不能由系统自动猜测，必须由人选择具体规格。
- 回库时沿用原出库项的产品和库位信息，回库数量加回对应库存。

## 7. AI 仓库查询口径

简尚 AI 查询仓库时必须按下面方式回答：

- 同名多规格分开说。
- 必须说明规格、仓库单位、当前库存、低库存线、库位。
- 不把 5L、20kg 当成库存单位。
- 不在信息不完整时编造库位或库存。

正确回答示例：

```text
霞光沙目前有两种规格：
1. 霞光沙5L：仓库单位桶，当前15桶，低于3桶提醒，库位A-1-1-1。
2. 霞光沙1L：仓库单位桶，当前20桶，低于3桶提醒，库位A-1-1-2。
```

错误回答示例：

```text
霞光沙库存35L。
```

原因：35L 不是仓库单位，也混淆了 1L 和 5L 两种规格。

## 8. 新材料录入检查清单

录入前：

- 确认材料是否已经存在。
- 如果存在同名材料，确认规格是否相同。
- 确认仓库单位是不是“桶、支、卷、个”等真实盘点单位。
- 确认是否有正式库位编码。

录入时：

- 名称、规格、仓库单位不要混在一个字段里。
- 库存数量只填数字。
- 低库存提醒填数字。
- 单价按仓库单位填写，例如 500/桶。
- 临时位置不要写入正式库位。

录入后：

- 搜索材料名称，看同名多规格是否都能显示。
- 搜索库位编码，看该位置材料是否能找到。
- 检查页面显示是否一眼能看懂。
- 如用于自有库存供货或项目出库，确认下拉能选到正确规格。

## 9. 常见问题

### 9.1 霞光沙 1L 和 5L 要不要分两条？

要。规格不同，库存不能混在一起。

### 9.2 仓库单位应该填 L 还是桶？

填桶。1L、5L 是规格；一桶才是仓库里拿货、盘点、扣库存的单位。

### 9.3 没整理好库位怎么办？

可以先不填库位，但不要用“门口左边”“地上”这类临时位置占正式字段。等仓库整理后再补 `A-1-1-1` 这种编码。

### 9.4 Excel 里写的是“阀值”，系统认吗？

认。系统会把 `阀值/阈值/低库存提醒/最低库存` 都当作低库存提醒。

### 9.5 同名材料导入时系统能自动判断规格吗？

只能在名称、规格、单位足够明确时自动匹配。否则必须人工确认，不能让系统猜。

## 10. Agent 接手注意事项

Codex、Claude、Hermes 后续处理仓库相关任务时默认遵守：

- 先保留真实业务口径，不为了技术方便改成员工看不懂的字段。
- 涉及库存扣减、回库、盘点确认时必须检查权限和审计日志。
- 自有库存路线必须绑定 `product_id`；总部直发路线不扣自有库存。
- UI 可以简化展示，但数据库字段不能再把规格、单位、库存混成一团。
- 如新增导入模板，必须先做表头预检，再写字段映射。
- 如修改 AI 仓库工具，必须同步“规格不是单位”的口径。
